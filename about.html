OUR MISSION

The Cosmos ecosystem is one of the most technically ambitious blockchain networks ever built ‚Äî a living internet of sovereign chains connected through the Inter-Blockchain Communication protocol. Yet most explorers serving this ecosystem are opaque, centralized, and require trusting a third party's data pipeline before you can understand what's happening on-chain.

Atom Registry was built to fix that. Our goal: give every developer, validator, delegator, and curious human a real-time window into the Interchain ‚Äî one that runs entirely in their browser, queries public nodes directly, and never passes your activity through our servers. We physically cannot log your searches because there is no server to log them on.

We believe blockchain infrastructure should be transparent by default. An explorer that is itself a black box is a contradiction in terms.

TECHNOLOGY

Atom Registry is a single HTML file. No build toolchain, no npm install, no framework. This was a deliberate decision: fewer moving parts means a more auditable, more trustworthy explorer.

DATA SOURCES

  • Cosmos SDK LCD/REST API ‚Äî cosmos, bank, staking, gov, distribution, slashing modules
  • Tendermint RPC via the REST gateway ‚Äî block, tx, validator queries
  • CosmWasm smart contract API ‚Äî code list, contract info, state queries
  • IBC core API ‚Äî channels, connections, clients, denom traces
  • CoinGecko public API ‚Äî price data and 24h sparklines

ENDPOINT STRATEGY

  • Per-chain curated endpoint lists with automatic failover rotation
  • Exponential backoff on failure ‚Äî 1s, 2s, 4s, 8s, up to 30s
  • Baked-in capability flags per chain eliminate unnecessary probe requests
  • Public Node and Cosmos Directory as primary fallback providers

CLIENT ARCHITECTURE

  • Vanilla ES6 JavaScript ‚Äî zero dependencies, zero framework overhead
  • CSS custom properties for the complete design system and theming
  • localStorage for cross-session caching of validators, blocks, and governance
  • Hash-based routing for deep-linkable pages and proposal modals
